Abstract

Text-based memory breaks over long horizons and wastes the context window. We instead use hypernetworks to allow an agent to edit their own parameters at inference time. A hypernetwork (Doc-to-LoRA) converts context into LoRA adapters through a single forward pass, avoiding the lengthy training runs of SFT or RL. We show three applications across memory, personalization, and skills, each with a live, interactive demo below.

Long-horizon memory

Every K steps, the oldest turns are compacted into an adapter and evicted from context. Memory persists in weights, not tokens, mitigating context window overflow.

Permanent personalization

During conversations, user preferences are updated in a document and periodically internalized as per-user LoRA profiles. Profiles persist across sessions without needing to reload context.

Self-improving skills

On failure, the agent reflects on its mistakes and writes a sharper task prompt. Text-to-LoRA converts it into a task adapter, specializing the agent's weights without traditional post-training.

One forward pass instead of a training run

SFT and RL adapt a model with long training loops. A hypernetwork instead predicts the adapter weights directly from an input, so adaptation happens in real time, mid-conversation.
